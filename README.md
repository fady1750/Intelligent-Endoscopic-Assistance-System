# 🔬 Intelligent Endoscopic Assistance System
### Medical Equipment II — SBE3220 · Task 02 · Premium Final Release

> A full-stack endoscopic simulation platform featuring real-time AI-assisted imaging analysis, multi-modal processing pipelines, and an always-on live monitoring interface — built for clinical training and research prototyping.

---

## 📁 Project Structure

```
.
├── endoscopic_premium_tabs.html   # Premium tabbed frontend (always-on live imaging)
├── backend.py                     # FastAPI backend — frame analysis & processing engine
├── requirements.txt               # Python dependencies
└── test_backend.py                # Automated backend test suite
```

---

## ⚡ Quick Start

### 1 — Install dependencies

```bash
pip install -r requirements.txt
```

### 2 — Launch the backend

```bash
python backend.py
```

### 3 — Open the application

```
http://127.0.0.1:8000/
```

The frontend loads automatically. No separate build step required.

## 🧩 Feature Overview

### 🖥️ Always-On Live Imaging Monitor
A persistent live endoscopic feed is embedded across **every tab** — clinical context is never lost while navigating the interface.

### 📂 Tab Workspace

| Tab | Purpose |
|---|---|
| **Live Imaging** | Primary feed with full imaging controls |
| **Dashboard** | System status, metrics, and session summary |
| **Navigation** | D-pad, keyboard, rotation, depth tracking, body map & 3D viewer |
| **Processing** | Real-time image processing pipeline |
| **AI Assist** | Backend-powered tissue analysis and risk assessment |
| **Gallery & Log** | Captured frames, procedure log, and printable report |

---

### 💡 Illumination Control
- Power level adjustment
- Intensity fine-tuning
- Spectral mode selection (white light, NBI, autofluorescence, etc.)

### 🎛️ Imaging Control
- Tissue type presets
- Zoom and focus sliders
- Frame capture with timestamp
- Measurement overlay toggle

### 🧭 Navigation Control
- On-screen D-pad and keyboard mapping
- Rotation axes (pitch, yaw, roll)
- Real-time depth tracking
- Interactive body map
- 3D viewer with canvas fallback

### ⚙️ Processing Pipeline

| Filter | Description |
|---|---|
| Noise Reduction | Smoothing and artifact suppression |
| Contrast Enhancement | Adaptive histogram equalization |
| Color Normalization | White balance and channel correction |
| Sobel Edge Detection | Anatomical boundary highlighting |
| Red-Zone Segmentation | Hemorrhage / inflammation flagging |
| Texture Mapping | Mucosal surface characterization |

### 🤖 AI Assist (Backend Analysis)
Each captured frame is sent to the Python backend for analysis:

| Parameter | Detail |
|---|---|
| Brightness & Contrast | Exposure quality metrics |
| Shape Detection | Lesion geometry analysis |
| Color Analysis | Chromoendoscopy-style tissue profiling |
| Texture Features | Surface pattern classification |
| Risk Level | Low / Moderate / High flagging |
| Recommendation | Actionable clinical suggestion |

## 🧪 Testing

Run the full test suite before submission or deployment:

```bash
# Backend unit tests
pytest -q test_backend.py

# Syntax validation
python -m py_compile backend.py test_backend.py

# Frontend script validation
node --check frontend_script_check.js
```

> **Note:** `frontend_script_check.js` can be regenerated by extracting the inline `<script>` block from `endoscopic_premium_tabs.html` if the file is missing.

You can also run the **built-in frontend self-test** directly from the UI via the self-test button in the interface.

## 📦 Dependencies

All Python dependencies are listed in `requirements.txt`. Key packages include:

## 📌 Notes

- All imaging data is **simulated** — no real patient data is used or stored.
- The system is designed as an **educational prototype** for SBE3220 coursework.
- Backend and frontend communicate over `localhost` — no external network required.
