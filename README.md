### ⚙️ Processing Pipeline

| Filter | Description |
|---|---|
| Noise Reduction | Gaussian blur (`radius = noise_strength / 3.0`, range 0–8) |
| Contrast Enhancement | Pillow contrast + sharpness boost (`contrast_strength`, range 0–100) |
| Color Normalization | `autocontrast` with adaptive cutoff (`color_norm`, range 0–100) |
| Sobel Edge Detection | Anatomical boundary highlighting |
| Red-Zone Segmentation | Hemorrhage / inflammation flagging via red-channel thresholding |
| Texture Mapping | Mucosal surface characterization via Laplacian variance |

## 🤖 AI Analysis Pipeline

### Classification
The backend classifies each frame into one of four tissue states:

| Label | Trigger condition |
|---|---|
| `Normal` | No significant indicators detected |
| `Polyp` | Largest contour area > 5.5 % of frame |
| `Lesion` | Edge density > 0.18 **and** Laplacian variance > 500 |
| `Bleeding` | Red-channel ratio > 8.5 % of pixels |

A `tissue_hint` field can be passed from the frontend to override classification directly (accepted values: `polyp`, `lesion`, `bleeding`, `normal`).

### Risk Scoring

Risk is computed as a cumulative score across five factors:

| Factor | Points |
|---|---|
| Classification is Bleeding or Lesion | +2 |
| Classification is Polyp | +1 |
| Red ratio > 6 % | +2 |
| Red ratio > 2.5 % | +1 |
| Edge density > 0.16 | +1 |
| Brightness out of range (< 45 or > 210) | +1 |

| Score | Risk Level | Recommendation |
|---|---|---|
| ≥ 4 | 🔴 High | Pause, clean lens, document frame, request clinical review |
| 2–3 | 🟡 Medium | Capture extra views, adjust illumination and inspect carefully |
| 0–1 | 🟢 Low | Continue routine navigation |

### Analysis Engine
The backend auto-selects its analysis engine at startup:

- **OpenCV** (preferred) — Canny edge detection, `findContours`, Laplacian for precise metrics
- **Pillow + NumPy** (fallback) — used automatically if OpenCV is unavailable; contour count is reported as `-1`

The active engine is returned in every `/api/analyze-frame` response as `"analysis_engine": "opencv"` or `"analysis_engine": "pillow-numpy"`.

## 🌐 API Reference

All endpoints are served on `http://127.0.0.1:8000`.

### `GET /api/health`
Returns backend status, active version, OpenCV availability, and a Unix timestamp.

```json
{
  "status": "online",
  "version": "3.0.0-premium-live-all-tabs",
  "opencv": true,
  "timestamp": 1714000000.0
}
```

---

### `POST /api/analyze-frame`
Analyzes a single captured frame. Accepts PNG or JPEG as base64 (raw or with `data:` prefix).

**Request body:**
```json
{
  "image_base64": "<base64 string>",
  "tissue_hint": "normal",
  "metadata": {}
}
```

**Response includes:** `classification`, `risk_level`, `recommendation`, image dimensions, per-channel means, edge density, contour count, red ratio, Laplacian variance, and texture level.

---

### `POST /api/process-frame`
Preprocesses a frame through the backend pipeline and returns the processed image alongside extracted features.

**Request body:**
```json
{
  "image_base64": "<base64 string>",
  "noise_strength": 2,
  "contrast_strength": 35,
  "color_norm": 12,
  "tissue_hint": null,
  "metadata": {}
}
```

**Response includes:** `processed_image_base64` (PNG), `mime_type`, `features`, and `metrics`.

---

### `POST /api/report`
Generates a structured session report ready for export or submission.

**Request body:**
```json
{
  "operator": "Dr. Smith",
  "procedure": "Colonoscopy",
  "patient_id": "PT-001",
  "elapsed_time": "00:14:32",
  "findings": "One polyp identified at 45 cm.",
  "metrics": {}
}
```

- All imaging data is **simulated** — no real patient data is used or stored.

- Backend and frontend communicate over `localhost` only — no external network calls are made.
- OpenCV is listed in `requirements.txt` but the backend **degrades gracefully** if it fails to import.
